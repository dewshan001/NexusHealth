# NexusHealth portability (copy to another PC)

This project is already mostly portable:
- It uses the Maven Wrapper (`mvnw`, `mvnw.cmd`) so you don’t need Maven installed.
- It uses SQLite at `jdbc:sqlite:${user.dir}/clinic.db`, so the database can live inside the project folder.

The only important rule is: **start the app while your working directory is the project root**, so `${user.dir}` points to the folder that contains `clinic.db`.

## What to copy
Copy the **entire** project folder (recommended: zip it first), including:
- `src/`
- `.mvn/`, `mvnw`, `mvnw.cmd`
- `pom.xml`
- `clinic.db` (if you want the exact same data)

Notes:
- `target/` is generated by Maven; it does not need to be copied.
- `clinic.db` is ignored by git, so **cloning the repo will not include your data**.

## Requirements on the new computer
- **Java/JDK 21** installed

## Run (Windows)
Option A (easiest):
- Double click or run: `run-portable.cmd`

Option B (PowerShell):
- Run: `run-portable.ps1`
  - If PowerShell blocks scripts, run once:
    - `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned`

## Run (macOS/Linux)
From the project folder:
- `./mvnw spring-boot:run`

## If the database file is missing
If `clinic.db` was not copied, the app will create a new SQLite file on first start.
To add the default test users, use one of:
- Run the app and create users via the signup UI, OR
- Use the SQL script:
  - `sqlite3 clinic.db < add-test-users.sql`

## Quick verification after copying
From the project folder:
- Check Java: `java -version`
- Check wrapper: `./mvnw -v` (or `mvnw.cmd -v` on Windows)
- Run tests: `./mvnw test`
